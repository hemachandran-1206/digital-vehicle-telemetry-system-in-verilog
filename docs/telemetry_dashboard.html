telemetry_topAll inputsAll outputsTop-level wiring: accel_final = slope_accel + control_accel
Signal Flow: position_in → position_reg → delta_calc → velocity_calc → delta_calc → accel_calc → hill_slope_calc → telemetry_top → acceleration_final
Cruise Control: tom_cruise_calc reads velocity and compares to target_speed, outputting control_accel to maintain target speed.
Slope Compensation: hill_slope_calc adjusts raw acceleration by slope_angle (uphill reduces, downhill increases).
Final Equation: acceleration_final = acceleration_raw − slope_effect + control_accel

Velocity & Position

Velocity tracks how fast the car moves per TIME_STEP. Position shows cumulative displacement. The overspeed limit (100) is marked in red. Scenario 3 shows velocity spiking to 120, triggering the overspeed alert.

Acceleration & Slope

Final acceleration = physics acceleration + cruise control + slope effect. Negative values shown in red. At 220ns (uphill), acceleration drops by ~10. At 260ns (downhill), acceleration rises by ~10. The harsh brake limit (−20) is marked in red.

Fuel & Distance

Fuel accumulates proportionally to velocity (fuel += velocity + 1). Distance counts total position changes (absolute value). Both increase monotonically over time.

Safety Alerts

Alert flags: bit[1]=Overspeed (velocity > 100), bit[0]=Harsh Brake (acceleration < −20). Scenario 3 (100−120ns) triggers overspeed alerts. Scenario 4 (140−160ns) triggers harsh brake alerts during emergency stop.

GTKWave Waveform Viewer

How to view signed values in GTKWave: Right-click on any signal (e.g., acceleration_final) → Data Format → select Signed Decimal or Decimal. By default GTKWave shows hex values where negative numbers appear as large hex strings (e.g., −5 = 0xFFFFFFFB). This dashboard shows signed decimal values for consistency.
